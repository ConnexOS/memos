# v0.4.x 版本特性

## 统一反馈/复用排序加成（v0.4.6）

`reuse_count` 替代 `useful_feedback_count` / `not_useful_feedback_count`。排序加成公式改为 `math.log2(reuse_count + 1) * 0.15`，移除 `_compute_feedback_boost()`。

## 设置面板三级折叠布局（v0.4.6）

Tab 布局 → 三级折叠卡片：端点管理(P0/必配) → 基础设置(P1/常用) → 高级设置(P2)，system_suggestion 独立 Tab 整合至高级区。

## i18n 国际化基础设施（v0.4.6）

`src/memos/i18n.py` — Translator 类 + JSON 翻译文件。Jinja2 模板注册 `_()` 函数，导航栏中/英切换按钮。

## 复用频率加成（Reuse Boost）

`_compute_reuse_boost()` — 被频繁检索的记忆获得评分加成：`boost = reuse_count × exp(-reuse_decay × days_since) × reuse_weight`，受 `reuse_boost_cap` 上限约束（默认 0.5）。`MemoryConfig` 中 `reuse_weight`/`reuse_decay`/`reuse_boost_cap` 控制，设 `reuse_weight=0` 可禁用。

## 自动 VACUUM

`_maybe_vacuum()` — 删除占比 >20% 且 >100 条时自动触发 SQLite VACUUM 回收空间。`_vacuum_lock` 互斥锁防止 VACUUM 期间并发写入致数据损坏。

## 质量评分与冲突检测（v0.4.1）

`MemoryConfig` 新增字段：`quality_threshold`、`conflict_detection_enabled`、`conflict_distance_threshold`、`expiry_warn_days`。当前为配置层预备，完整 UI 流程在 Dashboard 侧。

## 今日回顾 / 开发日报（`daily_review.py`）

从 ChromaDB 对话记录自动生成开发日报。核心流程：

1. **预清洗规则引擎**（零 LLM 成本）：短内容丢弃（<100字符）、纯命令丢弃、代码块压缩、滑动窗口重复合并
2. **策略选择**：根据清洗后轮次+LLM 可用性，自动选 DIRECT/PRE_SUMMARIZE/BATCH/PLAIN_LIST
3. **LLM 提炼**：调用 `MemoryExtractor._request_llm()` 生成结构化日报，含三级回退
4. **持久化**：写入 `document/日报/YYYY-MM-DD-开发日报.md`，支持追加和去重

CLI: `memos today`，Dashboard: `/api/conversations/daily-review`

## LLM 用量统计（`usage.py`）

`UsageLogger` — JSONL 追加写入（`etc/usage_log.jsonl`）+ 内存聚合查询。Dashboard `/api/usage/stats` 按 today/week/month 聚合。

## 分层检索与主动建议（v0.4.4）

`_build_layered_context()` 按相似度将检索结果分两层：

- **Layer 1（上下文注入）**: `sim >= 0.55`，注入 `additionalContext`，最多 3 条
- **Layer 2（主动建议）**: `sim >= 0.75`，写入 ChromaDB (`type=suggestion`)，Dashboard 展示

Layer 2 三重闸门：全局开关 → `.claude/no_suggestions` 免打扰文件 → 冷却期+每日上限。

MemoryConfig 新增 7 字段。可设 `MEMOS_USE_OLD_CONTEXT=1` 回退到旧版。Dashboard 新增 `/api/suggestions/*` 路由组（8 个端点）。

## 冲突检测增强（v0.4.5）

`_detect_conflicts_async()` 支持 `conflict_use_llm=false`：纯向量判断标记冲突。新增 `conflict_role` metadata（trigger/matched）。API 升级为配对返回格式，Dashboard 冲突弹窗升级为 VS 对比布局。

## 待办全生命周期管理（v0.4.5）

独立 `todo_status` 字段（pending/in_progress/completed/cancelled）。后端 API：列表/新建/编辑/状态流转/删除。Dashboard 待办面板支持分组+筛选+排序+拖放。MCP 工具 9→11（`list_todos`+`update_todo`），CLI 新增 `memos todo`。
